# Resumen Técnico — Sistema Senderos Tannat

## Arquitectura General

```
Landing Page (Vercel)
    ↓ POST JSON
n8n (Orquestador)
    ↓ API
Pagar.me (Pagamentos)
    ↓ Polling cada 5min
n8n (Confirmação)
    ↓
Evolution API (WhatsApp)
```

---

## Fluxo 1 — Recepção de Reserva e Criação de Pedido

**Trigger:** POST do `index.html` para `https://n8n.axion380.com.br/webhook/evento-reserva`

**Nós em sequência:**

**1. Webhook Entrada** — recebe o payload do formulário com `first_name`, `last_name`, `email`, `phone`, `country`, `document`, `quantity`, `amount`, `currency`, `event`

**2. Validar + Build Payload** — nó Code que:
- Valida campos obrigatórios
- Calcula `unitPrice = (amount / quantity) * 100` convertendo para centavos
- Detecta `country_code` do telefone pelo prefixo do número (não pelo país do selector)
- Constrói `orderPayload` completo para Pagar.me v5 com `billing_address` hardcoded do venue
- Gera `orderId` único: `TANNAT-{country}-{timestamp}-{rand4}`
- Monta `tableRow` para inserção PENDING
- Gera `auth_header` via `Buffer.from(sk_xxx + ':').toString('base64')`

**3. Tabela — Inserir PENDING** — DataTable insert com todos os campos da reserva em status `PENDING` antes de chamar Pagar.me

**4. Pagar.me — Criar Pedido** — HTTP POST para `https://api.pagar.me/core/v5/orders` com `fullResponse: true` e `neverError: true`

**5. Checkout OK?** — IF que verifica `statusCode < 400`

**6a. Responder OK → Frontend** — retorna `{ status: 'success', redirect_url, order_id }`

**6b. Extrair Erro Pagar.me** — Code que lê `body.message` e `body.errors[]` e monta mensagem legível

**6c. Responder Erro → Frontend** — retorna `{ status: 'error', message }` com HTTP 400

**Dados críticos salvos em metadata do pedido Pagar.me:**
```json
{ "source": "landing_tannat", "phone": "+59895435644", "country": "BR" }
```

## Fluxo 2 — Polling Pagar.me + Entrega do Ticket

**Trigger:** Schedule a cada 5 minutos

**Nós em sequência:**

**1. A cada 5 minutos** — ScheduleTrigger

**2. Preparar Credenciais** — Code que constrói `auth_header` e `api_url` com `?status=paid&size=20`

**3. Buscar Pedidos PAID** — HTTP GET para `https://api.pagar.me/core/v5/orders?status=paid&size=20`

**4. Filtrar Pedidos Tannat** — Code que filtra apenas pedidos com `metadata.source === 'landing_tannat'` e retorna um item por pedido

**5. Tem pedidos?** — IF que verifica `skip !== true`

**6. Buscar na Tabela** — DataTable GET com `returnAll: true` — traz todas as filas

**7. Gerar Ticket + QR + Mensagem** — Code que:
- Busca o `order` do nó `Filtrar Pedidos Tannat` ou `Dados de Teste`
- Filtra a fila correta com `allRows.find(r => r.json.order_id === orderId)`
- Verifica se `ticket_code` já existe — se sim retorna `skip: true`
- Gera `ticketCode = 'TKT-' + crypto.randomBytes(4).toString('hex').toUpperCase()`
- Gera URL do QR via `api.qrserver.com`
- Monta mensagem WhatsApp formatada

**8. Já processado?** — IF que verifica `skip !== true`

**9. Tabela — Atualizar PAID** — DataTable Upsert com `matchingColumns: order_id`, `defaultMatch: true`, atualiza apenas `status`, `ticket_code`, `paid_at`

**10. WhatsApp — Enviar Texto** — Evolution API nó nativo, instância `Maya`, envia mensagem formatada

**11. WhatsApp — Enviar QR** — Evolution API nó nativo, operação `Enviar Imagem`, envia QR com caption do ticket code

**Rama de teste manual:**
`Manual Trigger → Dados de Teste → Buscar na Tabela` (salta o polling)

## Fluxo 3 — Frontend (Landing Page)

**Stack:** HTML + Tailwind + JS vanilla deployado em Vercel

**Arquivos:**
- `index.html` — landing com modal de checkout
- `app.js` — lógica do formulário, lê de `window.CONFIG`
- `config.js` — configuração centralizada (preços, URLs, IDs)
- `vercel.json` — rewrites para `/success` e `/cancel`

**Fluxo UX:**
```
Botão "Reservar" → Modal com campos → POST a n8n → redirect_url → Pagar.me Checkout → Pix/Cartão → success.html
```

**Payload enviado ao n8n:**
```json
{
  "event": "tannat_cordero_sp",
  "first_name", "last_name", "email", "phone",
  "country", "document", "quantity", "amount", "currency"
}
```

---

## Tabela n8n — pagarme reservas

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `order_id` | string | Chave de matching — `TANNAT-{country}-{ts}-{rand}` |
| `full_name` | string | Nome completo do comprador |
| `email` | string | Email |
| `phone` | string | Telefone com prefixo internacional |
| `country` | string | País selecionado no formulário |
| `amount` | number | Valor em centavos |
| `quantity` | number | Quantidade de entradas |
| `currency` | string | `BRL` |
| `status` | string | `PENDING` → `PAID` → `CANCELED` |
| `ticket_code` | string | `TKT-XXXXXXXX` — gerado após pagamento |
| `paid_at` | string | ISO timestamp da confirmação |
| `created_at` | string | ISO timestamp da reserva |

## Pendências Abertas

**1. Webhook Pagar.me** — dashboard com bug 412. Solução temporária: polling cada 5 min. Email enviado para `suporte@pagar.me` solicitando registro manual.

**2. Upsert cria linha nova** — quando `order_id` não existe na tabela. Solução: garantir que o Fluxo 1 sempre insere antes do Fluxo 2 processar.

**3. `success.html`** — URL configurada em `SUCCESS_URL` mas página precisa existir no repo Vercel.

**4. Documento para não-BR** — CPF obrigatório no Pagar.me mas uruguaios não têm CPF. Decisão de produto pendente.
